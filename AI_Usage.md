# AI Usage

This project used AI as a development assistant during planning, debugging, refactoring, and documentation.

## How AI was used

AI was used to help with:

- Understanding Noroff Social API endpoints
- Writing and refactoring vanilla JavaScript modules
- Improving accessibility with labels, buttons, and aria-live messages
- Creating commit message suggestions
- Reviewing assignment requirements
- Writing README content
- Acting as a documentation resource for concepts, APIs, and best practices

## What I was responsible for

I reviewed and tested all code before committing it.

I made the final decisions about:

- Project structure
- Feature order
- Styling choices
- Commit timing
- What code was accepted or rejected
- Debugging results
- Final implementation

## Example prompts

Examples of things I asked AI about:

- How do I use the Noroff Social API authentication flow?
- How can I split a large API file into smaller modules?
- How can I add JSDoc comments to my functions?
- How can I write a README for this project?
- How can I debug broken import paths after moving files?

## Limitations

AI suggestions were not used blindly.  
All code was tested locally before being committed.

Some AI suggestions were adjusted, rejected, or rewritten when they did not fit the project requirements or caused issues.
